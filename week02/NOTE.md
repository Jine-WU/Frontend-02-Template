### 学习笔记

#### 语言通识

##### 形式语言（乔姆斯基普系）—— 大多数编程语言都是形式语言

1. 0型 无限制文法
2. 1型 上下文相关文法
3. 2型 上下文无关文法
4. 3型 正则文法 —— 能被正则表达式直接描述的文法

##### 产生式 —— 巴科斯-诺尔范式(BNF)

- 用尖括号括起来的名称来表示语法结构名 —— <>
- 语法结构分成基础结构和需要其他语法结构定义的复合结构
  - 基础结构称**终结符**
  - 复合结构称**非终结符**
- 引号和中间的字符表示**终结符** —— "+"
- 可以有括号 —— "("")"
- *****表示重复多次
- **｜**表示或
- **+**表示至少一次

##### 语言的分类

1. 形式语言—— 用途
   - 数据描述语言—— JSON、XML、SQL等
   - 编程语言 —— Java、JavaScript、c等
2. 形式语言—— 表达方式
   - 声明式语言—— HTML、CSS等
   - 命令型语言—— JavaScript等

##### 编程语言的性质

1. 图灵完备性—— 所有可计算的问题都可用来描述的语言就是具有图灵完备性
   - 命令式——图灵机（goto、if和while）
   - 声明式——lambda（递归）
2. 类型系统
   - 动态类型系统和静态类型系统
   - 强类型与弱类型（类型转换时发生）
   - 复合类型（结构体、函数的签名）
   - 子类型
   - 泛型（逆变/协变）

#### JS类型

##### JavaScript的基本类型

- Number
- String
- Boolean
- Object
- Null
- Undefined
- Symbol

#### JS对象

1. 在设计类时，我们不应该受到语言描述的干扰
2. 在设计对象的状态和行为时，我们总是遵循“行为改变状态的描述”的原则。
3. 在JavaScript运行时，原生对象的描述方式非常简单，我们只需要关心原型和属性（包含行为、状态）两个部分
4. JavaScript中描述方式
   - 基于原型—— Object.create Object.setPrototypeOf Object.getPrototypeOf
   - 基于类—— new class extends

#### 个人总结

在本周课程中收获很大的一点，在语言通识部分中，通过从语言的分类、编程语言的性质以及产生式中去了解一门编程语言的产生，这是在大学课程中都不常见的切入方式，虽然其中有些知识较为生涩，尚未完全理解。但这种感觉很奇特。以前只是为了写代码而写代码，如今对编程语言有了更深层次理解，这对平时编程时思维逻辑的的转变很有帮助。

另一个较为深刻的收获来源练习中“人被狗咬了”。这个问题的痛点在于，当对于对象中的状态与行为理解不够深刻时，很容易会分不清人和狗中到底谁的状态发生了变化。通过这个练习加深了对对象的理解。并拓展思维，平时编程中，对于一些公共方法、组件的编写，更值得花大量时间的应该是对其状态和行为的分析。
